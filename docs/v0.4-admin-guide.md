# v0.4 Admin Guide

## Scope and architecture

Admin pages are presentation only; API routes validate `app_metadata.role=admin` again. Knowledge and cases follow draft → reviewed → published → archived policy, with soft delete and audit events.

## Flow, errors and security

Published content cannot be edited directly. Publication validates the full shared schema. Audit metadata is allowlisted and never includes payloads, credentials, SQL or provider content.

## Flags, verification and limitations

Admin is disabled by default. Mock tests validate transition and authorization logic. Real app_metadata setup, RLS and audit database reads need a Preview Supabase project.
