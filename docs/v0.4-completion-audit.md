# v0.4 完成度盘点

本文件根据当前源代码、API 路由、迁移与页面实现审阅，而非只按文件名判断。

## 已完成且可实际使用

- Feature flag 全关时的本地决策、历史与 PDCA 仍走 v0.3 localStorage 路径。
- `AnalyzeResponse`、PDCA、云端 mutation 和同步 metadata 皆有 Zod schema。
- `/api/cloud/reports`、`/api/cloud/pdca` 与同步路由具备未配置、未登入、输入错误和安全错误码处理。
- Auth Proxy 以 `getClaims()` 验证，管理身份只取 `app_metadata.role`。
- migration 创建 RLS、拥有者策略、published 内容读策略与匿名化 rate-limit RPC。
- `/sync` 可扫描本机记录、保存 wizard 状态、显示离线／配置与安全的统计。

## 完整实作并完成 mock 验证

- Auth UI、站内 callback guard、account/logout、claims guard 与 user_metadata 拒绝逻辑。
- Cloud API、local-first repository、sync metadata、planner、25 笔 batch、partial result summary、retry 项目与 conflict resolver。
- Migration Wizard 的扫描、选择、取消／恢复状态与进度资料模型。
- Admin 共用元件、列表搜索／分页、状态转换、Admin API、audit metadata allowlist 与 audit UI。
- ContentRepository selector：旗标关闭只读 local；cloud 失败使用 `local_fallback`；成功空集合维持 `cloud` 空结果。
- RLS migration、只读审计 SQL、HMAC persistent rate-limit selector 与 memory fallback。
- 207 unit/integration 与 26 Playwright E2E mock／feature-flag-off regression。

## 尚待真实 Supabase 验证

- migration 与 seed 执行、Auth email / callback、真实 RLS、multi-device conflict、RPC rate-limit 与 admin app_metadata JWT。

## 暂以 mock 或本地实作

- 全部云端测试使用假 Supabase client／fetch；没有真实 project、key、migration 或 seed。
- rate limiter 会在无服务器 Secret 或数据库异常时降级至 memory。

## 需要 Supabase Project 才能实测

- Auth callback URL、Email 发送和 session 刷新。
- RLS policy、RPC grants、migration 与 seed。
- 多装置 revision conflict、持久化 rate-limit 与管理员 JWT `app_metadata`。

## 对 v0.3 的风险点

- Proxy 覆盖所有请求以更新 auth cookie，未配置 Supabase 时会直接 passthrough；必须通过回归 E2E 确认。
- `@supabase/*` 新依赖增加 bundle 与部署环境要求，但没有公开 Secret；Production flags 本 PR 保持关闭。
- Feature flag 默认关闭；无配置时不得改变本地报告或 PDCA 的存取。
