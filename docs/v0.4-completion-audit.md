# v0.4 完成度盘点

本文件根据当前源代码、测试、迁移与真实 Preview 数据库验证记录整理。

## 已完成并真实验证

- Feature flags 关闭时，本地决策、历史与 PDCA 仍走 v0.3 localStorage 路径。
- migration `20260715_wisdom_os_v04` 已套用；七张 public table 均启用 RLS。
- Policies、Data API table grants、function grants 与 SECURITY DEFINER search path 已完成真实验证。
- 内容 seed 已完成：`knowledge_entries` 56 笔、`case_entries` 30 笔。
- seed runner 直接读取 Supabase 原生 query result 的 `status`／`statusText`；每个 query 独立诊断，不使用 `captureStatus` wrapper 或 responses FIFO。已套用的 Admin/Audit hardening migration 进一步限制未来 system seed：只可插入缺少的 canonical published 内容；canonical／稳定 JSON 语义等价的相同列跳过，漂移、Admin 管理、非 published 与删除列全部 fail closed。Preview 的只读 preflight 已验证 knowledge 56／56、cases 30／30 相同，远端写入为 0。
- 云端 mutation、同步 metadata、Auth claims、Admin app_metadata、审计 metadata 和 rate-limit selector 都有 Zod／自动化覆盖。
- Protected Preview smoke test 已通过；同帐号跨装置下载已通过：云端一份报告与一轮 PDCA 成功还原为两笔本机资料，无效资料为零，同 ID 不覆盖本机。
- Account A／B 跨帐号 RLS 验收已通过：Account B 的 report／PDCA 清单均为零，无法读取、更新或删除 Account A 的临时验收资料；临时资料已清除，Account A 原有资料未修改。
- 385 个 unit/integration tests 与 27 个 Playwright E2E tests 已通过；20260719 已套用到 Preview，并完成 trigger、函数、grants、audit 安全边界与 canonical seed 的只读验证。

## 已完成但仍待真实浏览器验收

- Admin 状态机与 audit 原子性：additive migration `20260719_wisdom_os_admin_audit_hardening.sql` 已套用到 Preview，并确认 draft-only create、禁止 published／archived 原地写入、数据库 trigger 与最小化 audit event 的数据库结构均存在。真人 Account B Preview 验收已完成：普通角色与撤销角色后的新 JWT 均为 Admin API 403；临时 Admin 新 JWT 的完整 knowledge 生命周期、最小 case 生命周期、软删除和失败零写入均通过。Audit 总增量为 10（create 2、update 2、status_transition 4、soft_delete 2），unsafe metadata 与 system seed event 均为 0；两笔临时内容保留为软删除，公开 canonical 内容、Account A 资料与 report／PDCA 基线均未改变。
- persistent rate limit 的本机加固：单行原子窗口、HMAC IP 匿名化、严格 RPC 输入／回传验证、准确 health readiness 与数据库故障时 memory fallback。`20260720100901_wisdom_os_rate_limit_hardening.sql` 尚未套用 Preview，真人验收仍待独立批准。
