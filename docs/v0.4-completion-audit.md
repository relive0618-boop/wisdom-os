# v0.4 完成度盘点

本文件根据当前源代码、测试、迁移与真实 Preview 数据库验证记录整理。

## 已完成并真实验证

- Feature flags 关闭时，本地决策、历史与 PDCA 仍走 v0.3 localStorage 路径。
- migration `20260715_wisdom_os_v04` 已套用；七张 public table 均启用 RLS。
- Policies、Data API table grants、function grants 与 SECURITY DEFINER search path 已完成真实验证。
- 内容 seed 已完成：`knowledge_entries` 56 笔、`case_entries` 30 笔。
- seed runner 直接读取 Supabase 原生 query result 的 `status`／`statusText`；每个 query 独立诊断，不使用 `captureStatus` wrapper 或 responses FIFO。
- 云端 mutation、同步 metadata、Auth claims、Admin app_metadata、审计 metadata 和 rate-limit selector 都有 Zod／自动化覆盖。
- Protected Preview smoke test 已通过；同帐号跨装置下载已通过：云端一份报告与一轮 PDCA 成功还原为两笔本机资料，无效资料为零，同 ID 不覆盖本机。
- 309 个 unit/integration tests 与 27 个 Playwright E2E tests 已通过。

## 已完成但仍待真实浏览器验收

- 帐户 A／B 的 RLS 隔离与跨帐号资料不可见性。
- Admin `app_metadata.role=admin` 生效与 audit log。
- persistent rate limit 的 RPC、HMAC IP 匿名化与数据库故障时 memory fallback。

## Production 边界

Production credentials 与 flags 保持未设定；不启用 cloud UI 或 persistent rate limit。无 Supabase 配置时，应用仍应 build 成功并安全维持本地功能。

## 限制

Preview 的公开 smoke test 已在受保护 Preview 环境通过。该工具仅做 GET 验证，不登入、不写入资料、不读取 Secret。部署更新后，需人工确认还原报告的 PDCA 可查看，并确认刷新页面后报告与 PDCA 仍存在。
