# v0.4 Sync Protocol

## Scope and data flow

Scan local reports and cycles, fetch cloud metadata, create a plan, process batches of at most 25, then save safe local metadata. Operations are upload/create/update/delete, download/create/update, conflict or noop.

## Error and conflict handling

Each item produces a safe result. Partial failures remain pending for retry. Conflicts are never overwritten: retain local with expected revision, retain cloud with backup, retain both with a new ID, or defer.

## Security, flags and verification

Only hashes, IDs, revisions and validated payloads travel to the cloud. Feature flags off means no cloud request. Mock tests cover plan and batch behavior; multi-device behavior requires Preview verification.
