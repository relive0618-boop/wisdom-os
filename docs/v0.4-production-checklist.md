# v0.4 Production Checklist

## 已完成的 Preview 数据库基础

- Preview migration `20260715_wisdom_os_v04` 已真实套用。
- 七张 public table 的 RLS、Policies、table grants 与 function grants 已真实验证。
- 内容 seed 已真实验证：56 条 knowledge、30 条 cases。
- seed 使用 Supabase 原生 query result，保留原生 `status`／`statusText`，不使用 `captureStatus` 或 FIFO status 推测。

## Production 边界

本 PR 不启用 Production cloud features。Production 的 Supabase credentials 与 cloud flags 必须维持未设定或 false，且绝不可将 Preview Secret 复制过去。

## Production 前仍须完成的 Preview 验收

Preview 必须先完成真实 Auth、RLS 跨帐户、同步冲突、Admin app_metadata 权限、persistent rate limit 与 audit 验收。使用 `pnpm verify:preview -- --base-url <PREVIEW_URL>` 做只读公开 smoke test，并完成 `docs/v0.4-preview-live-verification.md` 的浏览器步骤。

Preview 启用 Deployment Protection 时，使用 `pnpm verify:preview -- --base-url <PREVIEW_URL> --vercel-protected`。此模式不会触及 Production，也不建立或输出 Protection bypass 凭证。

## 安全复核

确认浏览器 bundle、API response、日志与 localStorage 均不包含 Secret Key、rate-limit secret、JWT、cookie、raw IP、provider body 或 stack trace。Supabase 暂时故障时不得删除本地报告或 PDCA，Production 未配置时仍必须可成功 build 并维持本地模式。
