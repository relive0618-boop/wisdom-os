## Production 边界

本 PR 不启用 Production cloud features。Production 的 Supabase credentials 与 cloud flags 必须维持未设定或 false，且绝不可将 Preview Secret 复制过去。

## Production 前仍须完成的 Preview 验收

Preview 已完成受保护 smoke test、真实 Auth、同帐号跨装置下载与 Account A／B RLS 隔离：云端一份报告与一轮 PDCA 已成功还原为两笔本机资料，无效资料为零，同 ID 不覆盖本机；Account B 无法列出、读取、更新或删除 Account A 的验收资料，且临时资料已清除。Production 前仍必须完成真人 Admin `app_metadata` 权限、Admin／Audit mutation 与 persistent rate limit 验收。20260719 已以 `docs/v0.4-admin-audit-verification.sql` 完成 Preview 只读复核，但这不是真人 Admin mutation 验收。使用 `pnpm verify:preview -- --base-url <PREVIEW_URL>` 做只读公开 smoke test，并完成 `docs/v0.4-preview-live-verification.md` 的浏览器步骤。

当前 hardening 自动化验证为 385 个 unit/integration tests 与 27 个 Playwright E2E tests。20260719 已完成 Preview 只读验证，但不得标示为 Production 已验证。

Preview 启用 Deployment Protection 时，使用 `pnpm verify:preview -- --base-url <PREVIEW_URL> --vercel-protected`。此模式不会触及 Production，也不建立或输出 Protection bypass 凭证。

## 安全复核

确认浏览器 bundle、API response、日志与 localStorage 均不包含 Secret Key、rate-limit secret、JWT、cookie、raw IP、provider body 或 stack trace。Supabase 暂时故障时不得删除本地报告或 PDCA，Production 未配置时仍必须可成功 build 并维持本地模式。
