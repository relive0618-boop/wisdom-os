# v0.4 Production Checklist

## Scope

This PR does not enable Production cloud features. Keep all cloud flags false until Preview has passed Auth, RLS, sync conflict, rate-limit and audit verification.

## Manual Preview verification

Create a Supabase project, apply migration, seed published content, configure callback URLs and Preview variables, set an admin app_metadata role, then run the read-only RLS verification SQL.

## Security and limitations

Never copy Preview Secrets to Production. Confirm no key, token, raw IP or raw provider content appears in responses or logs. Persistent rate-limit and multi-device sync need real Project validation.
