# v0.4 Architecture

## Scope and architecture

Local report/PDCA storage remains authoritative on each device. Cloud repositories are optional adapters behind feature flags. Route handlers verify claims server-side and never trust a client user id.

## Data flow, errors and security

UI → repository → safe cloud API → RLS table. Every payload has Zod validation. Cloud errors retain local data. Server Secret and rate-limit HMAC never enter client components.

## Flags, mock verification and limitations

All flags default false. Unit and Playwright tests exercise local fallback and mocked cloud behavior. Preview needs a real Supabase Project to validate RLS; Production remains local mode.
